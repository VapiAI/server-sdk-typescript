* Several exported type constants and required interface fields have been removed. The following breaking changes require migration:
* Approximately 80 single-value provider/type enum constants (e.g. `ConversationNodeType`, `CustomLlmModelProvider`, `DeepgramTranscriberProvider`, `FallbackCustomTranscriberProvider`, `GroupConditionType`, and others) have been removed from the SDK's public exports. * The `provider` field has been removed from the `FallbackCustomTranscriber` interface. Remove any code that reads or assigns this property.
* The `type` discriminator field has been removed from the `GroupCondition` interface. Remove any code that reads or assigns this property.
* Several single-value provider and cost-type enums have been removed from the SDK (e.g. `HumeVoiceProvider`, `KnowledgeBaseCostType`, `VapiCostType`, `TranscriberCostType`, `VoiceCostType`, `SessionCostType`, `VoicemailDetectionCostType`, and many more). Additionally, the `type` discriminant field has been removed from the `KnowledgeBaseCost`, `LiquidCondition`, `SessionCost`, `SayPhoneNumberHookAction`, `TranscriberCost`, `VapiCost`, `VoiceCost`, and `VoicemailDetectionCost` interfaces. Consumers referencing these deleted symbols or reading the `type` field from those interfaces must update their code accordingly. The `BaseClientOptions.token` field has also been restructured into a `BearerAuthProvider.AuthOptions` intersection — verify client initialization if you pass `token` explicitly.
* The SDK now exposes a `fetch()` method on `VapiClient` for making passthrough HTTP requests that leverage the client's configured authentication, retries, logging, and base URL — useful for endpoints not yet natively supported.
* The `UpdateAssistantDtoCredentialsItem`, `UpdateAssistantDtoModel`, `UpdateAssistantDtoTranscriber`, and `UpdateAssistantDtoVoice` union types are now discriminated unions: each variant is a namespaced sub-interface with a required `provider` field (e.g. `UpdateAssistantDtoModel.Anthropic`). New providers have also been added across credentials (`AnthropicBedrock`, `Soniox`, `Email`, `SlackWebhook`), models (`AnthropicBedrock`, `Minimax`), and transcrbers (`Soniox`).
* The `callControllerFindAllPaginated` method has been removed from `CallsClient`. If your code calls `client.calls.callControllerFindAllPaginated(...)`, migrate to `client.calls.list(...)` or the equivalent listing method. Internal authentication handling has also been refactored to use a centralized `authProvider` pattern.
